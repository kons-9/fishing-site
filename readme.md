# 釣り記録サイト (Fishing Log Site) 概要

本プロジェクトは、釣りに関するあらゆる情報（道具の紹介、釣行日記など）を保存し、閲覧できるようにするための静的ウェブサイトです。GitHub Pagesを利用して公開することを前提としています。

## 目的
* 自身の釣り道具（タックル）を整理し、コレクションや備忘録として管理する。
* いつ、どこで、どんな条件で何が釣れたか（または釣れなかったか）を記録し、今後の釣行の参考にする。

## サイト構成とデータ定義

本サイトは主に「ホーム」「道具紹介」「釣り日記」の3つのコンテンツで構成されます。今後の実装（プランニング）に向けて、各コンテンツで管理する情報のデータ定義を以下にまとめます。

### 1. 道具紹介 (Gear)
所有している、あるいはおすすめの釣り道具を分類して紹介するページです。

**必要な情報（フィールド定義）:**
* **名前** (必須) : 道具の名称（例: ツインパワー C3000XG）
* **道具の分類** (必須) : ロッド、リール、ルアー、仕掛け、ウェア、小物など
* **対象の魚分類** (必須) : シーバス、アジ、青物、エギング、汎用など
* **メーカー / ブランド** (追加提案) : 道具のメーカー（シマノ、ダイワなど）
* **写真 / 画像** (追加提案) : 道具のビジュアル
* **スペック詳細** (追加提案) : 長さ、重さ、ルアーウェイト、糸巻量など
* **評価 / お気に入り度** (追加提案) : ★の数などで表す個人的な評価
* **レビュー / メモ** (追加提案) : 使用感や思い出、購入価格などのテキスト情報

### 2. 釣り日記 (Diary)
日々の釣行記録を保存するページです。過去のデータを分析しやすくするため、環境要因の記録も重視します。

**必要な情報（フィールド定義）:**
* **日付・時間帯** (必須) : 釣行した年月日と、時間帯（例: 5:00 - 10:00）
* **場所** (必須) : 釣行した釣り場、エリア
* **狙ったもの** (必須) : 本命のターゲット
* **釣果** (必須) : 釣れた魚種、サイズ、数（ボウズの場合はその旨も記録）
* **潮回り / 潮の流れ** (必須) : 大潮・中潮などの潮汐情報や、実際の潮の動き方（※「塩流れ」を潮汐情報として拡張）
* **天気・気温・水温** (追加提案) : 釣果に直結する環境要因
* **風向・風速** (追加提案) : 釣りのしやすさや魚の活性に影響するため
* **ヒットルアー / エサ** (追加提案) : その日釣れた具体的な仕掛けやルアー
* **使用タックル** (追加提案) : 「道具紹介」のデータと紐づけられると便利
* **写真 / 画像** (追加提案) : 釣れた魚や釣り場の風景写真
* **所感 / 反省点** (追加提案) : 次回への備忘録、その日のパターンの考察など

### 3. ホーム (Home)
サイトのトップページです。サイトの入り口として、最新の情報を表示します。

* サイトのタイトルと簡単な説明
* 最新の釣り日記（新着順に数件表示）
* 最近追加・更新された道具紹介（数件表示）

## 補足事項（別エージェント・実装設計者向け）
* このドキュメントはサイトの「要件」と「データモデル」を定義したものです。具体的な実装フレームワーク（Jekyll, Hugo, Next.js, Astroなど）やデザインの選定は別途プランニング（`plans.md`）にて決定してください。
* データの管理方法は、Markdownファイル（Frontmatter）による管理、あるいはヘッドレスCMSなどの導入を検討してください。GitHub Pagesでのホスティングを考慮したアーキテクチャを選定してください。
* 「道具」と「釣り日記（使用タックル）」の関連付け（リレーション）をどのように実現するかが、設計上のひとつのポイントになります。
